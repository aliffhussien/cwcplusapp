📑 Master Project Specification: CWC+ Media Command Center1. Core ObjectiveBuild a high-end, "Zero-Click" administrative media management module that allows even a non-technical user (or child) to manage images, names, and YouTube content for recipes and classes.2. Component A: Multi-Media Upload & Rename EngineInput Handling: Massive "Drop Zone" supporting batch uploads (JPG, PNG, WebP, HEIC).Background Logic:Auto-optimize images to WebP/AVIF.Generate sizes: Thumbnail (sq), Card (4:3), and Hero (16:9).The "Magic Rename" Logic (New):Inline Edit: Click directly on the filename label below the image to turn it into a text box.Auto-Sanitize: The system automatically replaces spaces with hyphens (-) and removes special characters in the background to prevent broken links.Visual Feedback: A small "Pencil" icon appears on hover to guide the user.Management UI:Visual grid of uploaded photos.Star Logic: Single click on a "Star" icon sets the Primary (Cover) image.Delete Logic: Simple "X" overlay on hover.3. Component B: YouTube BridgeMagic Link Input: A single field that accepts any YouTube URL.Automatic Extraction:Extract Video ID via Regex.Fetch maxresdefault.jpg as the local placeholder.Naming: Ability to rename the video title locally in the CMS without affecting the actual YouTube title.Safety Check: Built-in mini-player to verify the video within the CMS.4. Component C: "Empire" CMS UpgradesSmart Watermarking: Automatic "CWC+" logo overlay on the bottom-right of every image.SEO Schema Automator: Auto-generates "Recipe Schema" (JSON-LD) using the File Name as the ImageObject name. (This is why renaming is crucial!).Brand Color Extraction: Identifies the dominant color of the primary photo to subtly tint the app's UI.5. Design & UX (The "CWC+ Aesthetic")Style: Glassmorphism. Use background blurs ($12px+$), semi-transparent white borders, and soft shadows.Rename Animation: When clicking a name to edit, the text should "pop" into an input field using a spring animation.Simplicity: No "Save" buttons for renaming; it should auto-save when the user clicks away (Blur) or presses "Enter."6. Data Architecture (The Schema)Images: id, url, displayName, systemFileName, type: "image", isPrimary: boolean.Videos: id, yt_id, displayName, type: "video", thumbnail_url.Relational: All items must be tied to a parent ContentID.
